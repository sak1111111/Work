Вопросы для самоконтроля

1. Какие основные принципы REST архитектуры вы знаете?
   - Единообразие интерфейса (Uniform Interface)
   - Отсутствие состояния (Stateless)
   - Кэшируемость (Cacheable)
   - Клиент-серверная архитектура (Client-Server)
   - Многоуровневая система (Layered System)
   - Код по требованию (Code on Demand - опционально)

2. В чем разница между HTTP методами PUT и PATCH?
   - PUT используется для полного обновления ресурса (замена всего ресурса)
   - PATCH используется для частичного обновления ресурса (изменение только определенных полей)

3. Какие коды состояния HTTP наиболее часто используются в REST API?
   - 200 OK - успешный запрос
   - 201 Created - ресурс создан
   - 400 Bad Request - неверный запрос
   - 401 Unauthorized - не авторизован
   - 403 Forbidden - доступ запрещен
   - 404 Not Found - ресурс не найден
   - 500 Internal Server Error - внутренняя ошибка сервера

4. Что такое ресурсо-ориентированный дизайн URL?
   - Подход к проектированию URL, где каждый URL представляет собой ресурс
   - Использует существительные вместо глаголов (например, /users вместо /getUsers)
   - Иерархическая структура (например, /users/123/posts)

5. Какие преимущества предоставляет пагинация в API?
   - Снижение нагрузки на сервер
   - Уменьшение времени ответа
   - Экономия трафика
   - Улучшение пользовательского опыта
   - Предотвращение timeout'ов

6. Как организовать валидацию данных в REST API?
   - Валидация на стороне сервера
   - Использование схем валидации (JSON Schema, Joi, Yup)
   - Проверка обязательных полей
   - Валидация типов данных и форматов
   - Кастомные правила валидации

7. Какие подходы к обработке ошибок в REST API вы знаете?
   - Единый формат ответов об ошибках
   - Использование соответствующих HTTP статус-кодов
   - Детализация ошибок в теле ответа
   - Логирование ошибок на сервере
   - Глобальный обработчик ошибок

8. В чем разница между query parameters и path parameters?
   - Path parameters являются частью URL пути (/users/123)
   - Query parameters добавляются после знака ? (/users?page=1&limit=10)
   - Path parameters идентифицируют конкретный ресурс
   - Query parameters используются для фильтрации, сортировки, пагинации
