Вопросы для самоконтроля

1. Что такое Node.js и в чем его преимущества для создания серверов?
Node.js - это среда выполнения JavaScript, построенная на движке V8. 
Преимущества для создания серверов:
- Однопоточная событийно-ориентированная архитектура (Event Loop)
- Высокая производительность и масштабируемость
- Использование JavaScript как на клиенте, так и на сервере
- Богатая экосистема пакетов (npm)
- Асинхронная обработка запросов без блокировки

2. Как создать простой HTTP-сервер в Node.js?
const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Hello World!');
});

server.listen(3000, () => {
    console.log('Server running on port 3000');
});

3. В чем разница между методами GET, POST, PUT и DELETE?
GET - получение данных (идемпотентный, безопасный)
POST - создание новых данных (неидемпотентный)
PUT - полное обновление существующих данных (идемпотентный)
DELETE - удаление данных (идемпотентный)

4. Что такое middleware и для чего оно используется?
Middleware - это промежуточные функции, которые имеют доступ к объектам запроса (req), ответа (res) и следующей функции middleware в цикле запрос-ответ. 
Используются для:
- Логирования
- Аутентификации и авторизации
- Парсинга тел запросов
- Обработки ошибок
- CORS

5. Как обрабатывать различные HTTP-методы для одного URL?
С помощью фреймворков типа Express.js:
const express = require('express');
const app = express();

app.route('/api/users')
    .get((req, res) => { /* получить пользователей */ })
    .post((req, res) => { /* создать пользователя */ })
    .put((req, res) => { /* обновить пользователя */ })
    .delete((req, res) => { /* удалить пользователя */ });

6. Что такое CORS и зачем нужны соответствующие заголовки?
CORS (Cross-Origin Resource Sharing) - механизм безопасности браузера, который разрешает или запрещает запросы с других доменов.
Соответствующие заголовки нужны для:
- Разрешения кросс-доменных запросов
- Контроля доступа к ресурсам
- Указания разрешенных методов, заголовков и источников

7. Как организовать чтение тела POST-запроса в Node.js?
В нативном Node.js:
const http = require('http');

const server = http.createServer((req, res) => {
    if (req.method === 'POST') {
        let body = '';
        req.on('data', chunk => {
            body += chunk.toString();
        });
        req.on('end', () => {
            console.log(JSON.parse(body));
            res.end('Data received');
        });
    }
});

В Express.js с middleware:
app.use(express.json()); // для JSON
app.use(express.urlencoded({extended: true})); // для форм
