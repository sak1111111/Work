
1. Основные проблемы legacy PHP-кода:
SQL-инъекции (прямая подстановка данных в запросы)

Смешение логики и представления (HTML в PHP)

Отсутствие валидации пользовательского ввода

Использование устаревших mysql_* функций

Нет защиты от XSS и CSRF атак

2. Преимущества Laravel:
MVC архитектура для четкого разделения кода

Встроенная безопасность (CSRF защита, SQL-инъекции)

Blade шаблонизатор с наследованием

Миграции баз данных

Автоматическая загрузка классов

Eloquent ORM для работы с БД

3. MVC-архитектура в Laravel:
Model (app/Models) - работа с данными и бизнес-логика

View (resources/views) - представление (Blade шаблоны)

Controller (app/Http/Controllers) - обработка запросов, связь Model и View

4. Composer:
Менеджер зависимостей для PHP

Управление пакетами и библиотеками

Автозагрузка классов

Обновление зависимостей проекта

5. Возможности Blade:
Наследование шаблонов (@extends, @section)

Экранирование вывода (автоматическая защита от XSS)

Директивы (@if, @foreach, @while)

Подключение подшаблонов (@include)

Компиляция в чистый PHP

6. Маршрутизация в Laravel:
Маршруты определяются в файлах routes/

web.php - для веб-маршрутов

api.php - для API маршрутов

Поддержка GET, POST, PUT, DELETE методов

Параметры маршрутов: /user/{id}

7. Меры безопасности Laravel:
Защита от SQL-инъекций через Eloquent ORM

CSRF токены для форм

Экранирование вывода в Blade

Валидация данных

Хэширование паролей

Защита от XSS атак

8. Передача данных в шаблоны:
php
return view('template', ['key' => $value]);
return view('template')->with('key', $value);
return view('template', compact('variable'));
