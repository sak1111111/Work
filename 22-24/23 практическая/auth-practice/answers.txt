Ответы на вопросы для самоконтроля:

1. Разница между аутентификацией и авторизацией:
   - Аутентификация - проверка подлинности пользователя (кто ты?)
   - Авторизация - проверка прав доступа (что тебе разрешено?)

2. Преимущества JWT перед сессиями:
   + Stateless - не требует хранения на сервере
   + Масштабируемость
   + Кроссплатформенность
   - Невозможность отзыва до истечения срока
   - Больший размер

3. Важность хеширования паролей:
   - Защита при утечке БД
   - Предотвращение чтения паролей в чистом виде
   - Bcrypt использует salt и адаптивную стоимость

4. Salt в хешировании:
   - Случайные данные, добавляемые к паролю
   - Защита от rainbow table атак
   - Уникальность для каждого пароля

5. Обновление JWT токенов:
   - Access Token (короткоживущий) + Refresh Token (долгоживущий)
   - При истечении access token клиент использует refresh token для получения новой пары

6. Защита от brute-force:
   - Ограничение попыток входа
   - Временные блокировки
   - CAPTCHA
   - Увеличение задержки

7. Система ролей и разрешений:
   - Ролевая модель (user, moderator, admin)
   - Middleware проверки ролей
   - Иерархия прав доступа

8. Меры безопасности JWT:
   - Надежный секретный ключ
   - Ограниченное время жизни
   - HTTPS для передачи
   - Безопасное хранение на клиенте
   - Проверка подписи
